BUG-01 · SEQ Key Format Mismatch (SeqIncludeZone)
File: StingTools/Core/TagConfig.cs
Lines: GetExistingSequenceCounters() ~2345 · BuildTagIndexAndCounters() ~2410
────────────────────────────────────────────────────────────────────────────────
PROBLEM:
  GetExistingSequenceCounters() and BuildTagIndexAndCounters() both build the
  seqKey as:  string key = $"{disc}_{sys}_{lvl}";   (3-part, never includes zone)

  BuildAndWriteTag() builds seqKey as:

     string zone = ParameterHelpers.GetString(elem, ParamRegistry.ZONE);

         ? $"{disc}_{zone}_{sys}_{lvl}"
         : $"{disc}_{sys}_{lvl}";

2. In BuildTagIndexAndCounters() (~line 2410): same zone-read + conditional key.

3. Route both through the existing BuildSeqKey() static helper if it already
   handles the SeqIncludeZone conditional — otherwise apply fix directly.

VERIFY:
  - Create 3 elements with identical DISC/SYS/LVL but different ZONE.
  - Set SeqIncludeZone=true in project_config.json.
  - Run AutoTagCommand → assert 3 unique SEQ numbers (0001, 0002, 0003).

BUG-02 · Non-Overwrite Mode Silently Corrupts TAG1
File: StingTools/Core/TagConfig.cs
Lines: BuildAndWriteTag() lines 1561–1583
────────────────────────────────────────────────────────────────────────────────
PROBLEM:
  In non-overwrite mode, after SetIfEmpty runs (which correctly preserves
  existing LOC/ZONE), a validation loop fills any null slots in actualTokens[]
  with derived defaults, then unconditionally overwrites TAG1 with this hybrid.

  The dangerous code (lines 1563–1583):
    string[] actualTokens = ParamRegistry.ReadTokenValues(el);
    string[] defaults = { disc, loc, zone, lvl, sys, func, prod, seq };
    for (int i = 0; i < actualTokens.Length && i < defaults.Length; i++)
    {
        if (string.IsNullOrEmpty(actualTokens[i]))
            actualTokens[i] = defaults[i];   // ← injects derived value silently
    }
    tag = string.Join(Separator, actualTokens);   // ← hybrid tag assembled
    // then: ParameterHelpers.SetString(el, ParamRegistry.TAG1, tag, overwrite: true);

  Result: a manually-set LOC="BLD2" element gets TAG1 containing "BLD1" (the
  derived default) while ASS_LOC_TXT still correctly holds "BLD2".
  Compliance scan reads TAG1 → reports wrong data.

FIX:
  Remove the null-fill loop entirely from the non-overwrite path.
  If a token slot is empty after SetIfEmpty, leave it empty in the assembled tag.
  The existing malformed-tag guard at line 1577 (parts.Length < 8) correctly
  blocks incomplete tags from being written — let it do its job.

  REPLACE the loop:
    for (int i = 0; i < actualTokens.Length && i < defaults.Length; i++)
    {
        if (string.IsNullOrEmpty(actualTokens[i]))
            actualTokens[i] = defaults[i];
    }
  WITH: (nothing — delete the loop entirely for the non-overwrite path)

  Keep the defaults[] array for the overwrite path only (the existing
  `if (overwriteTokens)` branch at line 1539 — that branch is correct).

BUG-04 · AutoTagger Violates Revit API IUpdater Contract
File: StingTools/Core/StingAutoTagger.cs
Location: Execute() lines ~265–295
────────────────────────────────────────────────────────────────────────────────
PROBLEM:
  StingAutoTagger.Execute() calls Parameter.Set() (via PopulateAll and
  BuildAndWriteTag) inside an IUpdater.Execute() without an explicit
  Transaction. This violates the Revit API contract, corrupts the undo stack,
  and is the primary root cause of the auto-tagger auto-disabling after 3
  failures and worksharing sync crashes.

FIX — Refactor to ExternalEvent queue pattern:

STEP 1: Add to StingAutoTagger class:
  private static readonly System.Collections.Concurrent.ConcurrentQueue<ElementId>
      _pendingQueue = new System.Collections.Concurrent.ConcurrentQueue<ElementId>();
  private static ExternalEvent _autoTagEvent;
  private static AutoTagQueueHandler _autoTagHandler;

STEP 2: Create new inner class AutoTagQueueHandler : IExternalEventHandler:
  public string GetName() => "STING Auto-Tag Queue";
  public void Execute(UIApplication app)
  {
      Document doc = app.ActiveUIDocument?.Document;
      if (doc == null) return;
      var ctx = TokenAutoPopulator.PopulationContext.Build(doc);
      var (tagIndex, seqCounters) = TagConfig.BuildTagIndexAndCounters(doc);
      int processed = 0;
      using (var tx = new Transaction(doc, "STING Auto-Tag"))
      {
          tx.Start();
          while (processed < 50 && _pendingQueue.TryDequeue(out ElementId id))
          {
              Element el = doc.GetElement(id);
              if (el == null || !el.IsValidObject) continue;
              string catName = ParameterHelpers.GetCategoryName(el);
              if (string.IsNullOrEmpty(catName) || !TagConfig.DiscMap.ContainsKey(catName)) continue;
              TokenAutoPopulator.TypeTokenInherit(doc, el);
              TokenAutoPopulator.PopulateAll(doc, el, ctx, overwrite: false);
              TagConfig.BuildAndWriteTag(doc, el, seqCounters,
                  skipComplete: true, existingTags: tagIndex,
                  collisionMode: TagCollisionMode.AutoIncrement);
              string[] tokenVals = ParamRegistry.ReadTokenValues(el);
              ParamRegistry.WriteContainers(el, tokenVals, catName);
              processed++;
          }
          tx.Commit();
      }
      if (!_pendingQueue.IsEmpty) _autoTagEvent.Raise(); // drain remaining
      ComplianceScan.InvalidateCache();
  }

STEP 3: In StingAutoTagger.Register():
  _autoTagHandler = new AutoTagQueueHandler();
  _autoTagEvent = ExternalEvent.Create(_autoTagHandler);

STEP 4: In StingAutoTagger.Execute() (IUpdater.Execute) — REMOVE all
  PopulateAll/BuildAndWriteTag/WriteContainers/WriteTag7All calls.
  REPLACE with:
  foreach (ElementId id in data.GetAddedElementIds())
  {
      // discipline filter (read-only — allowed in IUpdater)
      Element el = doc.GetElement(id);
      if (el == null) continue;
      string cat = ParameterHelpers.GetCategoryName(el);
      if (string.IsNullOrEmpty(cat) || !TagConfig.DiscMap.ContainsKey(cat)) continue;
      _pendingQueue.Enqueue(id);
  }
  if (!_pendingQueue.IsEmpty) _autoTagEvent.Raise();

STEP 5: In Toggle(): wire/unwire _autoTagEvent alongside the IUpdater triggers.

BUG-05 · _paramCache Multi-Document Key Collision
File: StingTools/Core/ParameterHelpers.cs
Location: CachedLookup() ~line 82
────────────────────────────────────────────────────────────────────────────────
PROBLEM:
  The static _paramCache is keyed by (ElementId typeId, string paramName).
  ElementId values are document-relative integers — they collide across open
  documents. In multi-document Revit sessions, Document A's TypeId 1234 and
  Document B's TypeId 1234 share the same cache slot → wrong parameter
  Definitions returned → values written to wrong parameters silently.

FIX — Two-line change:

  FROM:
    private static readonly ConcurrentDictionary<(ElementId, string), Definition>
        _paramCache = new ConcurrentDictionary<(ElementId, string), Definition>();

    private static Parameter CachedLookup(Element el, string paramName)
    {
        ElementId typeId = el.GetTypeId();
        var key = (typeId, paramName);

  TO:
    private static readonly ConcurrentDictionary<(int docHash, ElementId typeId, string name), Definition>
        _paramCache = new ConcurrentDictionary<(int, ElementId, string), Definition>();

    private static Parameter CachedLookup(Element el, string paramName)
    {
        ElementId typeId = el.GetTypeId();
        var key = (el.Document.GetHashCode(), typeId, paramName);

  Also: in StingToolsApp.OnStartup(), subscribe to
  ApplicationEvents.DocumentOpened and call ParameterHelpers.ClearParamCache()
  in the handler (in addition to the existing DocumentClosed handler).

File: StingTools/Core/ParameterHelpers.cs
────────────────────────────────────────────────────────────────────────────────
FIX:
  In TokenAutoPopulator.PopulateAll(), BEFORE SpatialAutoDetect runs for LOC:

  private static bool ConnectorInherit(Document doc, Element el)
  {
      try
      {
          FamilyInstance fi = el as FamilyInstance;
          if (fi?.MEPModel?.ConnectorManager == null) return false;
          foreach (Connector conn in fi.MEPModel.ConnectorManager.Connectors)
          {
              if (conn.ConnectorType != ConnectorType.Physical) continue;
              var refs = conn.AllRefs;
              if (refs == null) continue;
              foreach (Connector other in refs)
              {
                  Element owner = other.Owner;
                  if (owner == null || owner.Id == el.Id) continue;
                  string otherTag = ParameterHelpers.GetString(owner, ParamRegistry.TAG1);
                  if (!TagConfig.TagIsComplete(otherTag)) continue;

                  ParameterHelpers.SetIfEmpty(el, ParamRegistry.LOC,
                      ParameterHelpers.GetString(owner, ParamRegistry.LOC));
                  ParameterHelpers.SetIfEmpty(el, ParamRegistry.ZONE,
                      ParameterHelpers.GetString(owner, ParamRegistry.ZONE));
                  ParameterHelpers.SetIfEmpty(el, ParamRegistry.SYS,
                      ParameterHelpers.GetString(owner, ParamRegistry.SYS));
                  ParameterHelpers.SetIfEmpty(el, ParamRegistry.FUNC,
                      ParameterHelpers.GetString(owner, ParamRegistry.FUNC));
                  return true;
              }
          }
      }
      catch (Exception ex) { StingLog.Warn($"ConnectorInherit: {ex.Message}"); }
      return false;
  }

  Call ConnectorInherit(doc, el) at the START of PopulateAll() before any
  other token population. If it returns true, spatial detection for LOC/ZONE

   public static string GetSidecarPath(Document doc)
   {
       string dir = OutputLocationHelper.GetOutputDirectory(doc);
       return Path.Combine(dir, Path.GetFileNameWithoutExtension(doc.PathName) + "_STING_SEQ.json");
   }

   public static void FlushSeqSidecar(Document doc, Dictionary<string, int> counters)
   {
       try
       {
           string path = GetSidecarPath(doc);
           File.WriteAllText(path, Newtonsoft.Json.JsonConvert.SerializeObject(counters));
       }
       catch (Exception ex) { StingLog.Warn($"FlushSeqSidecar: {ex.Message}"); }
   }

   public static Dictionary<string, int> LoadSeqSidecar(Document doc)
   {
       try
       {
           string path = GetSidecarPath(doc);
           if (!File.Exists(path)) return null;
           // Only use if sidecar is newer than the model file
           if (File.GetLastWriteTime(path) < File.GetLastWriteTime(doc.PathName)) return null;
           return Newtonsoft.Json.JsonConvert.DeserializeObject<Dictionary<string, int>>(
               File.ReadAllText(path));
       }
       catch { return null; }
   }

   load sidecar and merge (take max of sidecar vs scanned for each key):
   var sidecar = LoadSeqSidecar(doc);

           if (!seqCounters.ContainsKey(kvp.Key) || kvp.Value > seqCounters[kvp.Key])
               seqCounters[kvp.Key] = kvp.Value;

3. In ResolveAllIssuesCommand.cs, after each 500-element batch commit:

                 // Find all elements referencing this level and mark stale
                 string idStr = id.Value.ToString();
                 var stale = new FilteredElementCollector(doc)
                     .WhereElementIsNotElementType()
                     .Where(el => ParameterHelpers.GetString(el, "ASS_LVL_ELEM_ID_INT") == idStr)
                     .ToList();
                 using (var tx = new Transaction(doc, "STING Mark Level-Stale")) {
                     tx.Start();
                     foreach (var el in stale)
                         ParameterHelpers.SetString(el, "STING_STALE_BOOL", "1", true);
                     tx.Commit();
                 }
             }
         }
     };
