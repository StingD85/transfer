
## TIER 1 — INTELLIGENCE LAYER (TOKEN POPULATION)

### ITEM 1 — Token Inheritance from Family Type

**Files to read first:** `ParameterHelpers.cs` (TokenAutoPopulator class), `TagConfig.cs` (GetFamilyAwareProdCode)

**Problem:** When a BIM manager pre-populates `DISC`, `SYS`, `FUNC`, or `PROD` on a family TYPE in the project browser, those values are never pulled down to placed instances. TokenAutoPopulator reads only instance parameters and connector data — it ignores type-level token seeds entirely.

**Implementation:** Add a new static method `TypeTokenInherit(Document doc, Element el) → int` to `TokenAutoPopulator` in `ParameterHelpers.cs`:

```csharp
/// <summary>
/// Copies non-empty token values from the element TYPE to the instance,
/// only for tokens currently empty on the instance. Type-level values
/// are treated as higher-confidence seeds than auto-detection defaults.
/// Returns count of tokens inherited.
/// Call BEFORE PopulateAll() so inherited values are not overwritten.
/// </summary>
public static int TypeTokenInherit(Document doc, Element el)
```

Logic:
1. Get the element type: `Element typeEl = doc.GetElement(el.GetTypeId())`
2. For each token param in `{ DISC, SYS, FUNC, PROD, LOC, ZONE }`:
   - Read the value from `typeEl.LookupParameter(paramName)?.AsString()`
   - If non-empty AND the instance param is currently empty → write to instance via `ParameterHelpers.SetIfEmpty(el, paramName, typeValue)`
3. Return count written

Call `TypeTokenInherit(doc, el)` at the start of `TokenAutoPopulator.PopulateAll()` before all other token logic. Also call it inside `StingAutoTagger.Execute()` before `BuildAndWriteTag`.

### ITEM 2 — Cross-Element Token Copy ("Copy from Nearest")

**Files to read first:** `ParameterHelpers.cs` (SpatialAutoDetect), `TagConfig.cs` (GetElementCenter via SmartTagPlacement)

**Problem:** Runs of identical elements (20 VAV boxes on one level) each go through independent spatial detection, sometimes producing inconsistent LOC/ZONE values for nearly-identical positions. No mechanism exists to copy tokens from an adjacent already-tagged element of the same category.

**Implementation:** Add `CopyTokensFromNearest(Document doc, Element el, View view, string[] tokensToCopy) → int` to `TokenAutoPopulator`:

```csharp
/// <summary>
/// Finds the nearest already-tagged element of the same category in the same
/// level and copies the specified token values to el (only to empty params).
/// Uses a simple 2D distance check on element bounding box centers.
/// Falls back gracefully if no nearby tagged element exists.
/// Returns count of tokens copied.
/// Max search radius: 10 ft (model units). Only copies tokens listed in tokensToCopy.
/// </summary>
public static int CopyTokensFromNearest(
    Document doc, Element el, string[] tokensToCopy,
    IList<Element> candidatePool)
```

Algorithm:
1. Get `el`'s bounding box center (`bb.Min + bb.Max) / 2`)
2. Filter `candidatePool` to same category name AND same level ID AND `!string.IsNullOrEmpty(TAG1 param)` (already tagged)
3. Find the candidate with minimum XY distance to `el`'s center; skip if distance > 10 ft
4. For each `tokenName` in `tokensToCopy`: if nearest has non-empty value AND `el` has empty value → `SetIfEmpty`
5. Return count written

In `FullAutoPopulateCommand.Execute()`, after `TokenAutoPopulator.PopulateAll()` per element, call:
```csharp
CopyTokensFromNearest(doc, el, new[]{ ParamRegistry.LOC, ParamRegistry.ZONE, ParamRegistry.SYS }, allElements);
```

## TIER 2 — REAL-TIME AUTO-TAGGER ENHANCEMENTS

### ITEM 4 — Auto-Tagger Visual Tag Placement

**Files to read first:** `StingAutoTagger.cs` (Execute method ~line 157), `SmartTagPlacementCommand.cs` (TagPlacementEngine, PlaceTagsInView)

**Problem:** `StingAutoTagger.Execute()` writes data tags (parameters) only. No visual `IndependentTag` is placed. Users get ISO 19650 data but no annotation appears on screen.

**Implementation:** In `StingAutoTagger.Execute()`, after `BuildAndWriteTag()` succeeds:

```csharp
// Visual tag placement — only if user has enabled visual auto-tagging
if (_visualTaggingEnabled && doc.ActiveView != null
    && !(doc.ActiveView is ViewSheet)
    && doc.ActiveView.CanBePrinted)
{
    try
    {
        View view = doc.ActiveView;
        FamilySymbol tagType = TagPlacementEngine.FindTagType(doc, el.Category);
        if (tagType != null)
        {
            XYZ center = TagPlacementEngine.GetElementCenter(el, view);
            double offset = TagPlacementEngine.GetModelOffset(view);
            // Use view-type-aware preferred side
            int preferred = TagPlacementEngine.GetPreferredSide(el.Category?.Name ?? "", view);
            XYZ[] candidates = TagPlacementEngine.GetCandidateOffsets(offset);
            XYZ bestPos = center + candidates[preferred]; // fast: use preferred directly for real-time

            IndependentTag tag = IndependentTag.Create(
                doc, tagType.Id, view.Id, new Reference(el),
                false, TagOrientation.Horizontal, bestPos);

            // Set clean elbow
            if (tag != null)
            {
                try { tag.LeaderElbow = new XYZ(bestPos.X, center.Y, bestPos.Z); }
                catch { }
            }
        }
    }
    catch (Exception vex)
    {
        StingLog.Warn($"AutoTagger visual tag placement: {vex.Message}");
    }
}
```

Add `private static bool _visualTaggingEnabled = false;` to `StingAutoTagger`.

Add `public static void SetVisualTagging(bool enabled) { _visualTaggingEnabled = enabled; }`.

Add `AutoTaggerToggleVisualCommand` in `StingAutoTagger.cs`:
```csharp
[Transaction(TransactionMode.ReadOnly)]
public class AutoTaggerToggleVisualCommand : IExternalCommand
// Toggles _visualTaggingEnabled, shows current state in TaskDialog
```

### ITEM 5 — Auto-Tagger Workset Filter

**Files to read first:** `StingAutoTagger.cs`, Revit API `WorksetTable`, `WorksharingUtils`

**Problem:** In workshared projects the updater tags every placed element regardless of ownership, causing "element not owned" write failures and tagging elements belonging to other disciplines.

**Implementation:** Add a workset ownership check inside `StingAutoTagger.Execute()` before processing each element:

```csharp
// Workset filter: skip elements not owned by current user (workshared projects only)
if (doc.IsWorkshared)
{
    try
    {
        WorksetId wsId = el.WorksetId;
        if (wsId != null && wsId != WorksetId.InvalidWorksetId)
        {
            var wsInfo = WorksharingUtils.GetWorksharingTooltipInfo(doc, el.Id);
            // Skip if element is owned/checked out by another user
            if (!string.IsNullOrEmpty(wsInfo.Owner)
                && wsInfo.Owner != doc.Application.Username
                && wsInfo.Owner != "")
            {
                StingLog.Info($"AutoTagger: skipping el {id.Value} owned by '{wsInfo.Owner}'");
                continue;
            }
        }
    }
    catch { /* workset check is advisory — never block tagging */ }
}
```

Also add a discipline filter: read `_allowedDiscs` (a `HashSet<string>` that users can configure). If non-empty, only tag elements whose category maps to a DISC in `_allowedDiscs`. Add `public static void SetDisciplineFilter(IEnumerable<string> discs)` static method to `StingAutoTagger`.

Add `AutoTaggerConfigCommand` that shows a dialog letting the user configure both the workset filter (on/off) and the discipline filter (checkboxes for M/E/P/A/S/FP/LV).

### ITEM 6 — Linked Model Visual Tagging

**Files to read first:** `SmartTagPlacementCommand.cs` (PlaceTagsInView, FindTagType), Revit API `RevitLinkInstance`, `Reference.CreateLinkReference`

**Problem:** All placement commands skip elements in linked documents (silently increment `skipped` counter). On most projects, 30-60% of elements are in linked models.

**Implementation:** Add `PlaceTagsInLinkedViews(Document doc, View view, bool tagOnlyUntagged) → (int placed, int skipped, int collisions)` to `TagPlacementEngine`:

```csharp
public static (int placed, int skipped, int collisions) PlaceTagsInLinkedViews(
    Document doc, View view, bool tagOnlyUntagged)
```

Algorithm:
1. Collect all `RevitLinkInstance` elements visible in the view
2. For each link instance, open its document via `linkInst.GetLinkDocument()`
3. For each element in the linked doc that is visible in the view (use `FilteredElementCollector(linkDoc)` filtered to taggable categories):
   - Construct reference: `Reference linkRef = new Reference(linkInst).CreateLinkReference(elem)` — wrap in try/catch since API availability varies by Revit version
   - Find tag type using `FindTagType(doc, elem.Category)` — must use HOST doc, not link doc
   - Place `IndependentTag.Create(doc, tagTypeId, view.Id, linkRef, hasLeader, orient, tagPos)`
4. Linked tags are visual-only — never attempt to write parameters to linked elements

Add a separate `BatchPlaceLinkedTagsCommand` that wraps this method with the same scope/leader/preset dialogs as `SmartPlaceTagsCommand`. Register as `"BatchPlaceLinkedTags"` in WorkflowEngine and StingCommandHandler. Add to dock panel ORGANISE → Leaders section.

Add a clearly visible warning in the dialog: "Linked elements are read-only. Visual tags only — no parameter data written."

### ITEM 7 — Tag Cluster / Shared Leader for Dense Areas

**Files to read first:** `SmartTagPlacementCommand.cs` (TagPlacementEngine, SpatialGrid), `TagOperationCommands.cs`

**Problem:** Dense areas (panel boards, ceiling grids with 50 lights) produce illegible tag clouds even with collision avoidance. No mechanism for grouping nearby identical elements under a shared tag.

**Implementation:** Add `ClusterTagsCommand` in `TagOperationCommands.cs`:

```csharp
[Transaction(TransactionMode.Manual)]
[Regeneration(RegenerationOption.Manual)]
public class ClusterTagsCommand : IExternalCommand
```

Algorithm in Execute:
1. Scope: selected tags or active view
2. Grouping: cluster `IndependentTag` elements whose host elements are within `clusterRadius` (default: 3× base offset) of each other AND share the same category AND same DISC token
3. For each cluster of 2+ tags:
   - Keep the tag with the best (lowest-overlap) position as the representative
   - Delete all other tags in the cluster via `doc.Delete(tag.Id)`
   - Update the representative tag's display to show a count: write `STING_CLUSTER_COUNT` parameter (integer) on the representative element with value = cluster size
   - Append `"[×{count}]"` to the element's `PROD` token if `count > 1` (or write to a `STING_CLUSTER_LABEL` text param)
4. Report: N clusters formed, M tags consolidated

Also add `DeclusterTagsCommand` that reverses clustering by reading `STING_CLUSTER_COUNT` and re-running `SmartPlaceTagsCommand` on the clustered elements.

Add `STING_CLUSTER_COUNT` and `STING_CLUSTER_LABEL` to `ParamRegistry.cs` and `MR_PARAMETERS.txt` following the same pattern as Item 3.

Register: `"ClusterTags"` and `"DeclusterTags"` in WorkflowEngine. Add buttons to dock panel ORGANISE → Leaders section.

```csharp
public static int AlignTagBands(Document doc, View view, double tagHeightEstimate)
{
    var tags = new FilteredElementCollector(doc, view.Id)
        .OfClass(typeof(IndependentTag))
        .Cast<IndependentTag>()
        .OrderBy(t => { try { return t.TagHeadPosition.Y; } catch { return 0; } })
        .ToList();

    if (tags.Count < 2) return 0;

    // Group tags whose Y positions are within 1.2 × tagHeightEstimate of each other
    var groups = new List<List<IndependentTag>>();
    var current = new List<IndependentTag> { tags[0] };
    for (int i = 1; i < tags.Count; i++)
    {
        double prevY, thisY;
        try { prevY = tags[i-1].TagHeadPosition.Y; } catch { prevY = 0; }
        try { thisY = tags[i].TagHeadPosition.Y; } catch { thisY = 0; }

        if (Math.Abs(thisY - prevY) <= tagHeightEstimate * 1.2)
            current.Add(tags[i]);
        else
        {
            if (current.Count > 1) groups.Add(current);
            current = new List<IndependentTag> { tags[i] };
        }
    }
    if (current.Count > 1) groups.Add(current);

    int moved = 0;
    foreach (var group in groups)
    {
        var ys = group.Select(t => { try { return t.TagHeadPosition.Y; } catch { return 0.0; } }).OrderBy(y => y).ToList();
        double medianY = ys[ys.Count / 2];
        foreach (var tag in group)
        {
            try
            {
                XYZ pos = tag.TagHeadPosition;
                if (Math.Abs(pos.Y - medianY) > 0.001)
                {
                    tag.TagHeadPosition = new XYZ(pos.X, medianY, pos.Z);
                    moved++;
                }
            }
            catch { }
        }
    }
    return moved;
}
```

private static void RunSwitchTagPos(UIApplication app, int posValue)
{
    // Runs SwitchTagPositionCommand silently for active view scope,
    // preset position (no dialogs)
    try
    {
        var doc = app.ActiveUIDocument?.Document;
        if (doc == null) return;
        var view = doc.ActiveView;
        if (view == null) return;

        var elements = new FilteredElementCollector(doc, view.Id)
            .WhereElementIsNotElementType()
            .Where(e => e.Category != null && TagConfig.DiscMap.ContainsKey(e.Category.Name ?? ""))
            .ToList();

        var typeIds = new HashSet<ElementId>(elements.Select(e => { try { return e.GetTypeId(); } catch { return ElementId.InvalidElementId; } }).Where(id => id != ElementId.InvalidElementId));

        int updated = 0;
        using (Transaction tx = new Transaction(doc, $"STING Switch Tag Position {posValue}"))
        {
            tx.Start();
            foreach (ElementId typeId in typeIds)
            {
                try
                {
                    Element typeEl = doc.GetElement(typeId);
                    Parameter p = typeEl?.LookupParameter(ParamRegistry.TAG_POS);
                    if (p != null && !p.IsReadOnly) { p.Set(posValue); updated++; }
                }
                catch { }
            }
            tx.Commit();
        }
        StingLog.Info($"SwitchTagPos{posValue}: updated {updated} types in active view");
    }
    catch (Exception ex) { StingLog.Warn($"RunSwitchTagPos: {ex.Message}"); }
}
